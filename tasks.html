Homework 1: Tasks for kdh10

Your personal task is to develop a system for teams and their players.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Player. With the following private fields:

  1. String name
  2. String nationality
  3. String position
  4. int age
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Team. With the following private fields:

  1. int id
  2. String name
  3. String league
  4. List<Player> players
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Team> teams in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one team with two players. [4 marks]

Create controller co2103.hw1.controller.TeamController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/teams" should show view teams/list. [2 marks]
    • Add the list of all teams from co2103.hw1.Hw1Application to the model as attribute teams. [2 marks]
    • Create the corresponding JSP to list the name and league of each team followed by a link to URL "/players?team={id of team}", e.g., "/players?team=3". [4 marks]
  2. Requests for URL "/newTeam" should show view teams/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="team" and action="/addTeam" and inputs for all fields of class Team except for players. [5 marks]
  3. POST requests for URL "/addTeam" should add a new team to the list of all teams in co2103.hw1.Hw1Application and redirect to /teams. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.PlayerController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/players" should show view players/list. [2 marks]
    • Based on request parameter team determine the corresponding team from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all players of that team to the model as attribute players. [2 marks]
    • Create the corresponding JSP for the view to list the name, nationality, position, and age of each player. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newPlayer?team={id of team whose players are listed}", e.g., "/newPlayer?team=3". [4 marks]
  2. Requests for URL "/newPlayer" should show view players/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="player" and action="/addPlayer" and inputs for all fields of class Player. [6 marks]
    • Also add the request parameter team with the ID of the current team. [4 marks]
  3. POST requests for URL "/addPlayer" should add a new player to the list of players of the team identified by request parameter team and then redirect to /teams. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Team and make it available in controller TeamController. [4 marks]

When adding a new team validate that

  1. The value of field id is unique within the list Hw1Application.teams. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field league is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Player and make it available in controller PlayerController. [4 marks]

When adding a new player validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field nationality is not empty. [2 marks]
  3. The value of field position is either "Defence", "Attack", or "Goalkeeper". [3 marks]
  4. The value of field age is between (including) 18 and 90. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
